# 生命周期与恢复

当前现场只保留模块级启动入口，WebUI 通过 Manager 调用这些能力。不要用旧总测脚本或历史实验脚本启动机器人。

## 主运行入口

```text
start_base.sh
start_camera.sh
start_arm.sh
start_mapping.sh
start_navigation.sh
start_grasp.sh dry/live
start_web_console.sh
start_rviz_web.sh
stop_all.sh
```

## 维护入口

```text
start_handeye.sh
start_arm_keyboard.sh
run_arm_keyboard.sh
run_base_keyboard.sh
set_grasp_offset.sh
set_drop_point.sh
record_patrol_waypoint.sh
check.sh
diagnostics/check_robot_runtime.sh
diagnostics/collect_robot_snapshot.sh
```

## 恢复顺序

1. 先停止任务：`./scripts/stop_all.sh`。
2. 再检查锁状态：`./scripts/start_estop.sh status`。
3. 如急停锁存在，确认现场安全后执行：`./scripts/start_estop.sh reset`。
4. 按 `docs/模块启动说明书.md` 从底盘、相机、机械臂、导航、抓取逐模块恢复。

## 互斥规则

- `CALIBRATION` 手眼标定期间，不启动导航、抓取、WebUI 自动控制。
- `ARM_MANUAL` 机械臂手动控制期间，不启动抓取 live。
- `GRASP` 和 `NAVIGATION` 是主运行模式，由 Manager 或模块脚本持有。
- WebUI 不作为硬件 owner，只调用 Manager/Mission 服务。
