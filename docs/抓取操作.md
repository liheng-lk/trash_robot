# 感知与抓取

## Dry Run

只跑相机、VLM API 检测、深度定位、手眼变换、抓取决策，不让机械臂运动。相机由统一入口 `start_camera.sh` 管理，抓取脚本会自动确认相机已启动：

```bash
cd /home/sunrise/trash_robot_v3
export DASHSCOPE_API_KEY=你的阿里云百炼Key
./scripts/start_grasp.sh dry
```

视频：

```text
http://192.168.1.121:8092
```

检查目标：

```bash
set +u
source scripts/source_debug.sh
set -u
ros2 topic echo /trash_target_point_camera --once
ros2 topic echo /trash_target_point_arm --once
ros2 topic echo /trash_grasp_status
```

## Live

会启动机械臂服务并允许 `/trash_grasp_once` 真正运动。启动后不会自动抓取，自动抓取由任务主管或 WebUI 开关控制：

```bash
./scripts/start_grasp.sh live
```

单次抓取：

```bash
set +u
source scripts/source_debug.sh
set -u
ros2 service call /trash_grasp_once std_srvs/srv/Trigger "{}"
```

## 机械臂单独启动

```bash
./scripts/start_arm.sh
```

检查当前末端：

```bash
set +u
source scripts/source_debug.sh
set -u
ros2 service call /get_pose_cmd roarm_moveit/srv/GetPoseCmd "{}"
```

## 手眼标定

```bash
./scripts/start_handeye.sh
```

手眼标定同样复用统一相机入口，不再单独抢占 RealSense。
标定模式只开彩色图和 `camera_info`，不启动深度、对齐深度或点云。

浏览器打开：

```text
http://192.168.1.121:8093
```

输出文件：

```text
/home/sunrise/trash_robot_v3/config/grasp/handeye_point.yaml
```

开始采样前必须先过门禁测试：

```bash
./scripts/start_handeye.sh status
```

通过条件：

```text
image=true
camera_info=true
target_type=chessboard
target=true 或 chessboard=true
color image >= 10Hz
RealSense + handeye CPU <= 60%
```

如果棋盘格没有 OK，不要保存样本。当前棋盘格参数为内角点 `4x4`，
格子边长 `5mm`。每个样本只在画面稳定识别时保存，采 12-16 组，
覆盖近/远、左/右、高/低。
