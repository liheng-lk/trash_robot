# V3 项目结构

详细模块边界见 [模块边界与工具清单.md](模块边界与工具清单.md)。

## 主线

```text
建图: slam_toolbox
导航: Nav2
感知: VLM API 主线，本地模型后端后续接入
定位: D435i 深度图 + 手眼矩阵
执行: RoArm 分类抓取
交互: WebUI 控制台通过 Manager/Mission API 接入；RViz Web 使用真正 RViz2
```

## 目录职责

```text
config/      当前运行配置
docs/        操作、调试、维护文档
maps/        地图
scripts/     少量现场入口脚本
src/         ROS 2 主包和外部依赖包
tests/       静态结构和配置检查
runtime/     运行期日志、PID、临时状态；已用 COLCON_IGNORE 排除
```

顶层 launch 收敛到 `src/trash_robot_bringup/launch/`，根目录不保留 `launch/`。

## ROS 包边界

主运行包：

```text
trash_robot_manager
trash_robot_bringup
trash_robot_vision
trash_robot_grasp
trash_robot_mission
trash_robot_interfaces
trash_robot_web
```

外部依赖包：

```text
base_driver
sllidar_ros2
roarm_driver
roarm_moveit
roarm_moveit_cmd
roarm_description
moveit_servo
roarm_moveit_ikfast_plugins
```

外部依赖包只做配置适配，不承载垃圾分类业务逻辑。

## Scripts

`scripts/` 只保留现场入口、维护工具和公共库。当前脚本清单以 `scripts/脚本说明.md` 为准。新的复杂逻辑优先放到 ROS launch 或 node，脚本只负责启动入口。
