# Trash Robot V3 模块启动说明书

本文只写当前项目仍然有效的原始命令行启动方式。WebUI 是控制面，不是硬件启动真相；现场排障时优先按本文逐模块启动，确认每个模块的话题、服务和日志都正常，再交给 WebUI 或巡逻任务。

## 0. 基础环境

每次打开新 SSH 终端先执行：

```bash
cd /home/sunrise/trash_robot_v3
source scripts/source_v3.sh
```

只读检查：

```bash
./scripts/check.sh
./scripts/diagnostics/check_robot_runtime.sh
```

需要清空运行状态时：

```bash
./scripts/stop_all.sh
```

注意：`stop_all.sh` 会停止机器人运行组件并清理模式锁；不要在机器人正在执行抓取或导航时随手执行。

## 1. 底盘、雷达、TF

启动：

```bash
./scripts/start_base.sh start
```

状态：

```bash
./scripts/start_base.sh status
ros2 topic hz /odom
ros2 topic hz /scan
ros2 run tf2_ros tf2_echo odom base_link
```

停止：

```bash
./scripts/start_base.sh stop
```

正常现象：

- `/odom` 有发布；
- `/scan` 有发布；
- `odom -> base_link` TF 可查询；
- `/cmd_vel` 有底盘订阅者。

## 2. D435i 相机

启动完整相机链路：

```bash
./scripts/start_camera.sh start
```

状态：

```bash
./scripts/start_camera.sh status
ros2 topic hz /camera/camera/color/image_raw
ros2 topic hz /camera/camera/aligned_depth_to_color/image_raw
ros2 topic hz /camera/camera/color/camera_info
```

停止：

```bash
./scripts/start_camera.sh stop
```

手眼标定专用 color-only 模式：

```bash
./scripts/start_camera.sh handeye
```

注意：`rtp`/`srt` 是低延迟视频实验模式，会独占 D435i 的 V4L2 设备，不能和 ROS 相机同时运行。

## 3. 机械臂运行服务

启动：

```bash
./scripts/start_arm.sh start
```

状态：

```bash
./scripts/start_arm.sh status
ros2 service list | grep -E '/move_point_cmd|/get_pose_cmd|/set_gripper_cmd'
ros2 service call /get_pose_cmd roarm_moveit/srv/GetPoseCmd "{}"
```

停止：

```bash
./scripts/start_arm.sh stop
```

导航前把机械臂放到避障初始位：

```bash
./scripts/prepare_navigation_arm.sh start
```

当前导航初始位配置在脚本环境变量默认值中：

```text
x=118.1718722 mm
y=-0.725102626 mm
z=32.39033932 mm
```

## 4. VLM 识别与抓取链

只测试识别、深度、手眼和抓取计划，不运动机械臂：

```bash
./scripts/start_camera.sh start
./scripts/start_grasp.sh dry
./scripts/start_grasp.sh status
```

真实抓取前需要机械臂服务在线：

```bash
./scripts/start_camera.sh start
./scripts/start_arm.sh start
./scripts/start_grasp.sh live
./scripts/start_grasp.sh status
```

手动触发一次抓取：

```bash
ros2 service call /trash_grasp_once std_srvs/srv/Trigger "{}"
```

停止：

```bash
./scripts/start_grasp.sh stop
```

正常现象：

- `/vlm_trash_classifier` 在线；
- `/pixel_depth_locator` 在线；
- `/handeye_target_transformer` 在线；
- `/roarm_sort_grasper` 在线；
- `/trash_grasp_once` 服务在线；
- live 模式下 `/move_point_cmd` 与 `/get_pose_cmd` 在线。

模型接口配置：

```bash
sed -n '1,220p' config/perception/vlm_provider_registry.yaml
sed -n '1,180p' config/perception/vlm_trash_classifier.yaml
```

当前主线是 VLM API。本地模型后端只作为预留或显式 `local_hobot` 路径，不作为默认 fallback。

## 5. 建图

建图前先启动底盘和雷达：

```bash
./scripts/start_base.sh start
./scripts/start_mapping.sh start
```

保存地图：

```bash
./scripts/save_map.sh office_01
```

停止建图：

```bash
./scripts/start_mapping.sh stop
```

正常现象：

- `/slam_toolbox` 在线；
- `/map` 有发布；
- 保存后 `maps/office_01.yaml` 和对应图像文件存在；
- `runtime/current_map.txt` 指向最新保存地图。

## 6. Nav2 导航

启动：

```bash
./scripts/start_navigation.sh start
```

也可以指定地图：

```bash
./scripts/start_navigation.sh start /home/sunrise/trash_robot_v3/maps/344.yaml
```

初始化 AMCL 位姿：

```bash
./scripts/init_pose.sh <x> <y> <yaw_deg>
```

状态：

```bash
./scripts/start_navigation.sh status
ros2 action list | grep navigate_to_pose
ros2 topic echo /amcl_pose --once
ros2 topic hz /scan_depth
```

停止：

```bash
./scripts/start_navigation.sh stop
```

启动导航时脚本会做这些事：

1. 准备机械臂导航初始位；
2. 确认 `/odom` 和 `/scan`，必要时启动底盘；
3. 确认 D435i 深度，必要时启动相机；
4. 停止建图，避免两个 `/map` 来源冲突；
5. 启动 `depth_to_scan`；
6. 启动 Nav2；
7. 启动 `mission_supervisor`，供巡逻使用。

## 7. 巡逻点与巡逻任务

巡逻依赖导航和 AMCL。先在 RViz 或 RViz Web 中让激光点和地图墙体对齐，再记录点位。

记录第一条路线第一个点并清空旧点：

```bash
./scripts/record_patrol_waypoint.sh p1 --route office_loop --reset
```

继续追加点：

```bash
./scripts/record_patrol_waypoint.sh p2 --route office_loop
./scripts/record_patrol_waypoint.sh p3 --route office_loop
```

让已运行的任务主管重载路线：

```bash
ros2 service call /trash_mission/reload_route std_srvs/srv/Trigger "{}"
```

启动巡逻：

```bash
ros2 service call /trash_mission/start_patrol std_srvs/srv/Trigger "{}"
```

停止巡逻：

```bash
ros2 service call /trash_mission/stop_patrol std_srvs/srv/Trigger "{}"
```

如果服务不存在，先启动导航：

```bash
./scripts/start_navigation.sh start
```

## 8. WebUI 控制台

启动：

```bash
./scripts/start_web_console.sh start
```

访问：

```text
http://192.168.1.121:8095
```

状态和停止：

```bash
./scripts/start_web_console.sh status
./scripts/start_web_console.sh stop
```

WebUI 启动时会确保 Manager 和 Mission 控制面存在，但不应该绕过 Manager 直接抢硬件。硬件模块异常时，仍按本文对应模块命令排查。

## 9. 浏览器 RViz2

这是浏览器里的真正 RViz2，不是自制地图。

启动：

```bash
./scripts/start_rviz_web.sh start
```

访问：

```text
http://192.168.1.121:6080/vnc.html?autoconnect=1&resize=remote&path=websockify
```

状态和停止：

```bash
./scripts/start_rviz_web.sh status
./scripts/start_rviz_web.sh stop
```

如 noVNC 或 websockify 缺失，`start_rviz_web.sh` 会调用 `setup_rviz_web.sh` 安装到 `runtime/third_party/noVNC`。

## 10. 手眼标定

启动：

```bash
./scripts/start_handeye.sh start
```

访问：

```text
http://192.168.1.121:8093
```

状态和停止：

```bash
./scripts/start_handeye.sh status
```

```text
config/grasp/handeye_point.new.yaml
```

```bash

./scripts/trash_stack.sh camera start
./scripts/trash_stack.sh arm start
./scripts/trash_stack.sh navigation start
./scripts/trash_stack.sh grasp start dry

./scripts/trash_stack.sh status-all
./scripts/trash_stack.sh stop-all
```
