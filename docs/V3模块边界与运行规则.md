# V3 模块边界与运行规则

## V3 主运行链路

正式主线只保留：

```text
VLM识别 -> 深度定位 -> 手眼转换 -> 抓取投放 -> 导航巡逻 -> Web控制
```

当前识别默认使用 VLM API。本地 ROS 仍负责深度、手眼、安全窗口和机械臂动作。本地 BPU/COCO 模型不是当前阶段主线，后续作为可替换后端单独接入。

## 已清理内容

- 删除模型训练转换工具目录。
- 删除旧模型配置、旧标签映射和旧兜底说明。
- 删除旧视觉适配器执行入口和旧视觉 launch。
- 抓取脚本只接受 VLM API dry/live；传入 bpu/coco/mono2d/trash4 参数会明确失败，提示本地模型后续接入。

## 模块 Owner

| 模块 | 入口 | Owner 范围 | 不负责 |
| --- | --- | --- | --- |
| 底盘 | `./scripts/start_base.sh start` | 底盘驱动、雷达、基础 TF、`/odom`、`/scan` | Nav2、巡逻、机械臂、抓取 |
| 导航 | `./scripts/start_navigation.sh start` | Nav2、定位、巡逻主管 | 启动底盘、启动机械臂、启动标定 |
| 机械臂运行服务 | `./scripts/start_arm.sh start` | `/dev/roarm`、RoArm driver、MoveIt command 服务 | 键盘控制、抓取业务、WebUI |
| 手动机械臂工具 | `./scripts/start_arm_keyboard.sh start` | 人工维护调试、servo、键盘控制 | 自动抓取、巡逻 |
| 手眼标定工具 | `./scripts/start_handeye.sh start` | color-only 相机、标定 Web、标定样本 | 主 WebUI、导航、抓取 |
| 抓取 | `./scripts/start_grasp.sh dry/live` | VLM API、深度定位、手眼转换、抓取节点 | 启动相机、启动底盘、训练模型 |
| Web 控制台 | `./scripts/start_web_console.sh` | 用户入口、调用 Manager | 直接启动硬件、直接杀进程 |

## 相机 Owner 规则

- 主运行相机由 Manager 或 `start_camera.sh` 管理。
- 手眼标定只使用 color-only 相机。
- 视频流只订阅已有图像 topic，不直接打开 RealSense。
- 低延迟 RTP/SRT 是独立实验入口，会独占相机，不能和 ROS 相机同时运行。

## 串口 Owner 规则

- `/dev/roarm` 同一时间只能由一个模式使用。
- `start_arm.sh` 提供正式运行服务。
- `start_arm_keyboard.sh` 是人工维护工具。
- 抓取 live 依赖 `/move_point_cmd` 和 `/get_pose_cmd`，不直接打开串口。

## 模式锁互斥表

模式锁文件：`/tmp/trash_robot_v3_mode.lock`  
PID 目录：`/tmp/trash_robot_v3_pids`

| 请求模式 | 禁止同时存在 |
| --- | --- |
| `GRASP` | `ARM_MANUAL`、`CALIBRATION` |
| `ARM_MANUAL` | `GRASP`、`CALIBRATION` |
| `CALIBRATION` | `GRASP`、`NAVIGATION`、`VIDEO`、`ARM_MANUAL` |
| `NAVIGATION` | `CALIBRATION` |
| `VIDEO` | `CALIBRATION` |
| `WEBUI` | 不独占硬件 |

所有 stop 逻辑只停止自己记录的 PID，不使用宽泛进程匹配。

## dry/live 区别

- `dry`：启动 VLM API、深度定位、手眼转换、抓取状态，不执行机械臂真实运动。
- `live`：在 `dry` 基础上要求机械臂服务在线，允许执行真实抓取动作。

## 常用启动命令

```bash

./scripts/start_base.sh start
./scripts/start_camera.sh start
./scripts/start_navigation.sh start
```

## 验收命令

```bash
python3 -m compileall -q src/trash_robot_*
bash -n scripts/*.sh
bash -n scripts/lib/*.sh
pytest -q tests
colcon build

ros2 pkg executables trash_robot_vision
./scripts/start_grasp.sh dry bpu
./scripts/start_handeye.sh status
./scripts/start_arm.sh status
```

`./scripts/start_grasp.sh dry bpu` 预期失败，并提示当前阶段只启用 VLM API，本地模型后续接入。

## 禁止事项

- 不修改官方 SDK/vendor 包。
- 不覆盖 `config/grasp/handeye_point.yaml`。
- WebUI 不直接启动硬件、不直接 kill 进程。
- 禁止 `pkill -f ros2`、`pkill -f python`、`pkill -f trash_robot`、`pkill -f realsense`。
- 未经现场确认，不做真实底盘移动和机械臂运动测试。

## 已知问题

- WebUI 仍需要继续按 Manager 状态细分展示“机械臂服务在线”和“抓取节点在线”。
- 抓取点质量仍依赖 VLM 抓取点、深度稳定性、手眼标定和偏置实测。
- 标定输出先写入 `config/grasp/handeye_point.new.yaml`，需要人工确认后替换正式文件。
