# Trash Robot V3 模块边界与工具清单

原则：一个硬件只有一个运行 owner；维护工具不混入主闭环；WebUI 只能通过 Manager/Mission 受控入口接入。

## 主运行系统

| 模块 | 职责 | 入口 | 不负责 |
| --- | --- | --- | --- |
| 底盘/雷达 | `/odom`、`/scan`、基础 TF | `scripts/start_base.sh` | 业务导航决策 |
| 相机 | D435i 彩色、深度、对齐深度 | `scripts/start_camera.sh` | VLM 判断、抓取动作 |
| 机械臂运行服务 | RoArm driver、MoveIt、`/move_point_cmd`、`/get_pose_cmd` | `scripts/start_arm.sh` | 自动抓取策略 |
| 建图 | `slam_toolbox`、`/map` | `scripts/start_mapping.sh`、`scripts/save_map.sh` | Nav2 路径执行 |
| 导航 | Nav2、AMCL、depth_to_scan、mission_supervisor | `scripts/start_navigation.sh`、`scripts/init_pose.sh` | 抓取执行 |
| 抓取 | VLM API、深度定位、手眼转换、RoArm 分类投放 | `scripts/start_grasp.sh dry/live` | 启动底盘、建图 |
| 巡逻 | 点位路线、发现垃圾、抓取后恢复巡逻 | `trash_robot_mission` 服务 | 直接操作硬件驱动 |
| WebUI | 状态、按钮、视频、调用 Manager/Mission | `scripts/start_web_console.sh` | 直接启动硬件驱动、直接 kill 进程 |
| RViz Web | 浏览器里的真正 RViz2 | `scripts/start_rviz_web.sh` | 自制导航地图逻辑 |

## 独立工具

独立工具不属于主运行闭环，只用于标定、人工调试、点位记录和配置维护。


| 工具 | 入口 | 规则 |
| --- | --- | --- |
| 手眼标定 | `scripts/start_handeye.sh` | 进入 `CALIBRATION`，不和抓取/导航/WebUI 自动控制同时运行 |
| 机械臂手动控制 | `scripts/start_arm_keyboard.sh`、`scripts/run_arm_keyboard.sh` | 进入 `ARM_MANUAL`，不和抓取 live 同时运行 |
| 底盘手动控制 | `scripts/run_base_keyboard.sh` | 只在 `IDLE` 模式调试，禁止和导航/巡逻同时运行 |
| 巡逻点记录 | `scripts/record_patrol_waypoint.sh` | 只读取 `/amcl_pose` 写路线，不控制底盘 |
| 抓取/投放配置 | `scripts/set_grasp_offset.sh`、`scripts/set_drop_point.sh` | 修改配置后重启抓取链 |
| 诊断 | `scripts/check.sh`、`scripts/diagnostics/*` | 只读检查为主，不代替模块实测 |


## 外部依赖包

这些包是当前现场模块依赖，不作为独立业务入口随意删除：

| 包 | 用途 |
| --- | --- |
| `base_driver` | 底盘串口驱动，提供 `/odom`、订阅 `/cmd_vel` |
| `sllidar_ros2` | 雷达驱动，提供 `/scan` |
| `roarm_driver` | RoArm 串口硬件驱动 |
| `roarm_description` | 机械臂 URDF/描述文件 |
| `roarm_moveit` | MoveIt 配置、运动规划依赖 |
| `roarm_moveit_cmd` | `/move_point_cmd`、`/get_pose_cmd`、夹爪命令服务 |
| `moveit_servo` | 机械臂键盘/伺服手动调试依赖 |
| `roarm_moveit_ikfast_plugins` | MoveIt IK 插件依赖 |

## 已清理旧入口

旧 QR 任务管理、mock 抓取链、旧分层总测、Foxglove 观察入口、手眼 RTSP 实验入口、跨主机临时相机诊断均不属于当前现场启动链，已从 active 项目中删除。

## 当前模式

```text
IDLE
CALIBRATION
ARM_MANUAL
GRASP
NAVIGATION
```

硬规则：

- `CALIBRATION` 和 `ARM_MANUAL` 是维护模式。
- `GRASP`、`NAVIGATION` 是主运行模式。
- `WEBUI` 不作为硬件模式，只能调用 Manager/Mission。
- `common.sh`、`lib/*.sh` 是内部库，不直接作为现场入口。
