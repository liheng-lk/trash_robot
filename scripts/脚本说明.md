# 脚本说明

这份清单只列当前现场可用的脚本。脚本按功能模块保留，避免多个入口控制同一硬件资源。

## 推荐入口

```bash
source scripts/source_v3.sh
./scripts/trash_stack.sh help
./scripts/trash_stack.sh status-all
./scripts/trash_stack.sh stop-all
```

## 基础环境

| 脚本 | 用途 |
| --- | --- |
| `source_v3.sh` | 运行环境，设置 ROS2/CycloneDDS/项目 install 环境 |
| `source_debug.sh` | 调试环境，和 `source_v3.sh` 保持同一 DDS 策略 |
| `common.sh` | 内部公共函数，普通用户不直接执行 |
| `trash_stack.sh` | 统一模块入口，适合现场组合启动/停止/查看状态 |
| `lib/*.sh` | 内部模式锁、运动锁、栈控制库 |

## 主运行模块

| 模块 | 启动脚本 | 说明 |
| --- | --- | --- |
| 底盘/雷达 | `start_base.sh` | 启动底盘驱动、雷达和基础 TF |
| 相机 | `start_camera.sh` | 启动 D435i 彩色、深度、对齐深度；也提供 RTP/SRT 低延迟实验子命令 |
| 机械臂运行服务 | `start_arm.sh` | 启动 RoArm driver、MoveIt command 服务、位姿/移动/夹爪服务 |
| 抓取 | `start_grasp.sh` | 启动 VLM API、深度定位、手眼转换和分类抓取节点；支持 `dry/live` |
| 建图 | `start_mapping.sh`、`save_map.sh` | 启动 SLAM Toolbox 和保存地图 |
| 导航 | `start_navigation.sh`、`init_pose.sh` | 启动 Nav2、AMCL、深度避障 scan；发布初始位姿 |
| 巡逻点 | `record_patrol_waypoint.sh` | 从 `/amcl_pose` 记录路线点到 `config/mission/patrol_routes.yaml` |
| Web 控制台 | `start_web_console.sh` | 启动 FastAPI WebUI，并拉起 Manager/Mission 服务 |
| RViz Web | `start_rviz_web.sh`、`setup_rviz_web.sh`、`map_to_marker.py` | 浏览器访问真正 RViz2，保留 RViz 原生地图和 2D Nav Goal 行为 |

## 独立调试/维护工具

| 脚本 | 用途 |
| --- | --- |
| `start_handeye.sh` | 手眼标定入口，会进入 `CALIBRATION` 模式 |
| `start_arm_keyboard.sh`、`run_arm_keyboard.sh` | 机械臂键盘控制，会进入 `ARM_MANUAL` 模式 |
| `run_base_keyboard.sh`、`run_base_keyboard.py` | 底盘键盘控制，只允许 `IDLE` 模式下运行 |
| `set_grasp_offset.sh` | 查看/设置/微调抓取偏移 |
| `set_drop_point.sh` | 查看/设置/读取垃圾桶投放点 |
| `start_estop.sh` | 软急停、恢复、查看运动锁状态 |
| `check.sh`、`diagnostics/*` | 现场只读诊断和快照工具 |
| `stop_all.sh` | 停止本项目记录的运行组件并释放模式锁 |

## 保留原则

- 一个硬件资源只保留一个主入口：底盘、相机、机械臂、抓取、导航不要互相直接抢占。
- WebUI 只作为控制台，重活交给 Manager/Mission，不直接拥有硬件驱动。
- 抓取链路以 VLM API 为当前主线，本地模型只保留配置预留，不作为默认现场启动入口。
- 旧实验、旧验收、旧同步和临时 Agent 脚本不再属于 RDK 现场运行目录。
