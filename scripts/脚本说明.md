# 脚本说明

这份清单按“现场用户该不该直接运行”整理。原则是：`scripts/` 顶层只放现场运行和维护入口；训练、实验和一次性工具放到 `tools/`，避免误用。

## 最常用入口

```bash
source scripts/source_v3.sh
./scripts/trash_stack.sh help
./scripts/trash_stack.sh status
./scripts/trash_stack.sh status-all
./scripts/trash_stack.sh stop-all
```

## 普通现场用户直接运行

| 脚本 | 用途 |
| --- | --- |
| `trash_stack.sh` | 统一入口，推荐用它查看状态、启停模块 |
| `start_navigation.sh` | 启动底盘、相机、Nav2、AMCL、深度避障和 8092 视频 |
| `start_grasp.sh` | 启动 VLM 抓取链路，支持 `dry/live/status/stop` |
| `start_yolo_detector.sh` | 启动本地 YOLO 候选检测 |
| `start_web_console.sh` | 启动 WebUI 控制台 |
| `start_rviz_web.sh` | 启动浏览器版真实 RViz2 |
| `record_patrol_waypoint.sh` | AMCL 对齐后记录巡逻点 |
| `patrol_route_manager.sh` | 查看、切换、清空、设置巡逻路线模式 |
| `init_pose.sh` | 命令行发布 AMCL 初始位姿 |
| `save_map.sh` | 保存当前建图结果 |
| `stop_all.sh` | 停止项目内记录的所有运行组件 |
| `start_estop.sh` | 软急停、恢复、查看运动锁 |

## 维护/调试时使用

| 脚本 | 用途 |
| --- | --- |
| `start_base.sh` | 单独启动底盘、雷达、基础 TF |
| `start_camera.sh` | 单独启动 D435i，含 RTP/SRT 实验子命令 |
| `start_arm.sh` | 单独启动 RoArm driver 和 MoveIt 命令服务 |
| `start_mapping.sh` | 启动 SLAM Toolbox 建图 |
| `start_handeye.sh` | 手眼标定入口，会占用 `CALIBRATION` 模式 |
| `start_arm_keyboard.sh`、`run_arm_keyboard.sh` | 机械臂键盘控制，会占用 `ARM_MANUAL` 模式 |
| `run_base_keyboard.sh`、`run_base_keyboard.py` | 底盘键盘控制，只允许 `IDLE` 模式 |
| `set_grasp_offset.sh` | 查看/设置/微调抓取偏移 |
| `set_drop_point.sh` | 查看/设置/读取垃圾桶投放点 |
| `check.sh`、`diagnostics/*` | 现场诊断、快照、链路检查 |

| `patrol_route_manager.py` | `patrol_route_manager.sh` 的 Python 实现 |

## 已删除

| 脚本 | 原因 |
| --- | --- |
| `start_patrol_grasp.sh` | 旧组合入口，容易误导。巡逻抓取应由 `start_navigation.sh` 启动 Nav2/Mission，再由 `/trash_mission/start_patrol` 或 WebUI 控制；YOLO 和抓取链路分别由 `start_yolo_detector.sh`、`start_grasp.sh` 管理。 |

## 保留原则

- 一个硬件资源只保留一个主入口：底盘、相机、机械臂、抓取、导航不要互相直接抢占。
- 巡逻抓取由 Mission 状态机调度，不再用临时组合脚本拼接。
- WebUI 是控制台，重活交给 Manager/Mission，不直接拥有硬件驱动。
- 本地模型、训练、标注和一次性工具放入 `tools/`，不放在 `scripts/` 顶层。
